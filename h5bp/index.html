HTML5 ★ Boilerplate

멋진 HTML5 를 위한 견고한 기본 스타일.

처음 방문했어요: 우선 뭐가 좋은지 설명해 줘요.

HTML5 Boilerplate 는 프로페셔널한 프론트엔드 개발자의 빠르고 강력한 미래형 사이트를 만들기 위한 기본 HTML/CSS/JS 템플릿입니다.

4년이상의 반복적인 개발을 통해, 최고만을 엄선하여 모범사례를 만들었습니다: 크로스 브라우저의 정상적인 작동 지원, 성능 최적화, 크로스 도메인 XHR 과 Flash 관련 옵션도 제공해요. 아파치용 .htaccess 설정파일에는 캐싱 룰을 작성하여, HTML5 비디오, @font-face 에 대응하고 있습니다. 강력한 gzipping 으로 사이트의 파일사이즈를 절반으로 떨어뜨렸구요.

Boilerplate 는 프레임워크가 아닙니다. 어떠한 개발철학을 규정짓거나 하지 않고, 단지 여러분의 프로젝트를 쉽고 빠르게 진행할 수 있는 몇가지 트릭을 제공합니다.

좋은 점들

  • 크로스 브라우저 호환 (IE6+, 물론 지원해요.)
  • HTML5 의 새로운 태그들을 사용했음.
  • A등급 성능을 위한 최적화 캐싱과 압축 룰
  • 사이트 기본 설정에 대한 최고의 모범사례
  • 모바일 브라우저 최적화
  • 우아한 퇴보(graceful degradation)와 점진적 향상(Progressive enhancement) … 을 적용했어요
  • 크로스 브라우저 제어의 최대화를 위해, IE만의 클래스 사용
  • 간편한 .no-js 와 .js 클래스를 사용해서 기능에 맞춘 스타일
  • Console.log 로 브라우저가 다운되는 문제를 해결함.
  • doctype markup 은 완벽하므로, 절대 잘못될 수 없음!
  • 성능 최적화된 프린트 스타일 시트
  • iOS, 안드로이드, 오페라 모바일에 대응하는 마크업과 CSS 뼈대(skeleton).
  • .clearfix, .visuallyhidden 클래스에서 현명하고 접근성높은 스타일 적용.
  • HTML5 기능의 올바른 사용과 빠른 페이지로딩을 .htaccess 파일에 적용
  • CDN 호스팅의 jQuery 를 사용하고, 실패시 로컬에서 호출하도록 적용
  • 너무 많은 기능이 있나요? HTML5 Boilerplate 은 delete 키와 친해요. :)

HTML5 Boilerplate 는 3.0 이에요!

2012년 2월 6일, 월요일

이번 업데이트의 주요기능은 Boilerplate 를 보다 작게 만든 거에요. 대부분의 변경사항이 작게 만드는데 초첨을 맞췄어요. 우리의 중요한 작업을 위한 빌드스크립트를 우리의 저장소에 보관하기로 했어요. 그럼 이제 새로운 변경사항을 소개할께요.

변경내역

3.0 은 몇몇 주요 업데이트를 했어요. 이 링크에 마지막 릴리즈이후 커밋된 모든 내용을 담고 있어요.

기존 사이트를 업데이트해야할까요? 짧게 답하자면: 아뇨, 안해도 돼요.

주요 변경사항

  • 많은 폴더를 지웠어요: /demos, /test, 그리고 /js/mylibs.
  • 빌드스크립트는 별도의 프로젝트로 분리했어요, node.js 빌드스크립트 & ruby/rake 빌드스크립트 introduced. 빌드스크립트를 얻고 사용하는 방법에 대해 읽기
  • Respond.js 는 기본적으로 사용하지 않기로 했어요.

index.html

  • Chromeframe 은 스크립트로 적용하지 않기로 했어요. IE6 사용자는 Chromeframe 을 설치할 지, 아니면 Browse Happy 를 방문해서 다른 브라우저를 선택하게 할 지 링크를 표시합니다.
  • iOS 의 스케일링 버그(scaling bugs)를 막기 위해 Meta 속성의 viewport 를 업데이트했어요.
  • html 안의 IE 클래스는 명시적으로 지정된 버전보다 낮은 버전을 적용하도록 했어요.
  • Modernizr.load 대신 최적화된 Google Analytics 의 스니펫을사용하도록 했어요.
  • 오래된 IE 버그를 피하기 위해 모든 스크립트에서 defer 를 제거했어요.
  • 404 페이지의 디자인을 개선했어요.

style­.css

  • 보다 나은 기본환경을 제공하기 위해 기본폰트사이즈와 행간은 삭제했어요.
  • 보다 나은 타이포그래피를 제공하기 위해 normalize.css 를 추가했어요.
  • 보다 강력한 .ir 헬퍼 클래스.
  • jQuery UI 프로젝트와 다른 폭을 계산하는 라이브러리등에 영향을 주지 않도록 더이상 스크롤바를 강제하지 않아요.

프로젝트

  • Modernizr 2.5 로 업데이트
  • jQuery 1.7.1 로 업데이트
  • .htaccess 개선
